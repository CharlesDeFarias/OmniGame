# Cooking Game + Hub Implementation Plan (Plan 8)

> Subagent-driven; TDD for core/services; build-gates + review for scenes; branch `feat/cooking-hub`; commits as Charles DeFarias; controller merges/deploys. Decisions #44 (relaxed assembly, no timers), #48 (everyday basics), #35 (shared economy), near-zero text throughout.

**Goal:** Game #2 — relaxed real-recipe cooking (assembly loop) — plus the HubScene game picker that makes OmniGame an omnibus. Cooking earns the same coins/xp; recipes get "filmed" as cooking content (career synergy later).

**Fixed design:**
- Core (`src/core/cooking/`): pure TS, deterministic, ZERO Phaser. Recipe = ordered steps; Step union: `gather` (tap N required ingredients from a pantry that includes distractors; order-free), `sequence` (perform actions in order: chop/stir/pour/flip/spread/blend/cook), `assemble` (stack layers in order). Engine: `startRecipe(recipe)` → state; `applyInput(state, input)` → { state, correct, done, events } — wrong inputs NEVER fail: `correct:false`, mistake counter++. Stars: mistakes ≤1 → 3, ≤3 → 2, else 1. Events drive the renderer.
- Recipes (`src/core/cooking/recipes.ts`): 10 everyday basics, 3-5 steps each, escalating step counts: 01 buttered toast, 02 fruit salad, 03 sandwich, 04 scrambled eggs, 05 smoothie, 06 pancakes, 07 pasta with sauce, 08 quesadilla, 09 veggie soup, 10 mini pizza. Ingredient/action vocabularies as string-union ids. Realism matters (real ingredient sets, sensible order: crack eggs before stirring) — educational per spec.
- Progression service (`src/services/cooking.ts`, storage `omnigame.cooking.v1`): { version:1, best: Record<recipeId, stars>, unlocked: number } — recipe n+1 unlocks when n completed (any stars). Rewards on completion via wallet: coins 30+10*stars, xp stars*10, journal `recipe_done` {id, stars, mistakes}.
- CookingScene: recipe list view (grid cards: dish icon + star row + lock state) → play view per step type: gather = pantry grid (required + 4-6 distractors, tap correct → flies to bowl; wrong → wiggle); sequence = 2-4 large action buttons, correct order (pulse hint on next action after 6s idle); assemble = layer icons tapped in order stack visually on a plate. Finish: plated dish + camera flash + star pop + coin payout pips. All icons procedural (theme additions: ~24 ingredient + 7 action textures — flat, chunky, recognizable; log judgment calls). Near-zero text (numbers + icons only).
- HubScene (boot scene, always first): 'Luana Studio' ring-light logo motif, currency bar, two big gold-framed game cards — match-3 (gem icon → CareerScene) and cooking (pan icon → CookingScene) — plus 2 dimmed teaser cards (gate-runner, tower). Career gains a small home button (top-left) → hub; CookingScene likewise.
- Tutorial: first recipe (toast) shows the pointing hand on the first required ingredient / first action.
- Adaptive difficulty: NOT applied to cooking (relaxed by design). Dance-break counter unaffected.

**Tasks:** T0 core types+engine (TDD ~12 tests). T1 recipes data + validation test (every recipe's ingredients/actions exist in vocab, steps well-formed, escalation sanity). T2 cooking progress service (TDD) + wallet reward wiring design. T3 icon textures + recipe-card/hub textures. T4 CookingScene (list + 3 step renderers + finish flow). T5 HubScene + boot rework + home buttons + navigation. T6 final review (engine correctness probes, scene lifecycle, economy integration, journal), merge, deploy, docs.
